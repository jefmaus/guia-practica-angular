Glosario de Conceptos Clave

Los decoradores, conceptos y patrones más usados en Angular moderno

@ComponentCore
Decorador que convierte una clase TypeScript en un componente Angular. Define su selector HTML, template y estilos. Desde Angular 17, standalone: true es el valor por defecto — ya no necesitas NgModule.
💡 Cada componente = un archivo .ts + .html + .css. El selector es el nombre del tag HTML.
@Component({
  selector: 'app-usuario',
  standalone: true,
  imports: [NgOptimizedImage, DatePipe],
  template: `
    <h2>{{ nombre() }}</h2>
    <p>{{ creado | date:'dd/MM/yyyy' }}</p>
  `,
  styleUrl: './usuario.component.css',
  changeDetection: ChangeDetectionStrategy.OnPush
})
export class UsuarioComponent {
  nombre = signal('Ana García');
  creado = new Date();
}
@Injectable / ServiceDI
Marca una clase como inyectable por el sistema DI. Con providedIn: 'root' crea una única instancia global (singleton). Usar inject() en lugar del constructor es el patrón moderno.
💡 Los servicios son el lugar correcto para lógica compartida, HTTP y estado global.
@Injectable({ providedIn: 'root' })
export class UsuarioService {
  private http = inject(HttpClient);
  private url  = '/api/usuarios';

  getAll() {
    return this.http.get<Usuario[]>(this.url);
  }
  getById(id: number) {
    return this.http.get<Usuario>(`${this.url}/${id}`);
  }
}
@Input / @OutputComunicación
@Input() recibe datos del padre. @Output() emite eventos hacia el padre con EventEmitter. En Angular 17+ existen las funciones input() y output() basadas en Signals.
💡 Regla: los datos bajan con @Input, los eventos suben con @Output. El flujo siempre hacia arriba.
// Forma moderna con Signal Inputs (Angular 17+)
@Component({ selector: 'app-card', standalone: true, ... })
export class CardComponent {
  // input() retorna un Signal de solo lectura
  titulo   = input.required<string>();
  subtitulo = input('Sin subtítulo');

  // output() — sin EventEmitter
  seleccionado = output<number>();

  onSeleccionar(id: number) {
    this.seleccionado.emit(id);
  }
}
<!-- <app-card [titulo]="'Mi card'" (seleccionado)="fn($event)" /> -->
Standalone ComponentsAngular 17+
Desde Angular 17, los componentes son standalone por defecto — no necesitan NgModule. Importan sus dependencias directamente. El arranque se hace con bootstrapApplication() y providers funcionales.
💡 Todo proyecto nuevo usa standalone. NgModule existe pero es el modelo legado.
// main.ts — arranque moderno
bootstrapApplication(AppComponent, {
  providers: [
    provideRouter(routes, withComponentInputBinding()),
    provideHttpClient(withInterceptors([authInterceptor])),
    provideAnimationsAsync(),
    provideClientHydration(),  // SSR
  ]
});
inject()DI Moderno
Función de inyección de dependencias (Angular 14+). Reemplaza al constructor con parámetros. Solo puede usarse en contexto de inyección: al definir campos de clase, en el constructor, o en factory functions.
💡 inject() es la forma preferida. Es más limpio, composable y permite crear funciones reutilizables.
export class ProductoComponent {
  // Inyección directa en campos de clase
  private svc      = inject(ProductoService);
  private router   = inject(Router);
  private route    = inject(ActivatedRoute);
  private destroy  = inject(DestroyRef);
}

// Función reutilizable con inject()
function usePaginacion(paginaTam = 10) {
  const router = inject(Router);
  const pagina = signal(1);
  return { pagina, siguiente: () => pagina.update(p => p + 1) };
}
PipeTemplate
Transforma valores en el template usando el operador |. Hay pipes integrados (DatePipe, CurrencyPipe, AsyncPipe, JsonPipe) y se pueden crear personalizados. Las pipes puras solo recalculan cuando cambia la referencia del input.
💡 Usa async pipe para suscribirte a Observables — se desuscribe automáticamente al destruir el componente.
<!-- Pipes built-in -->
{{ fecha | date:'dd/MM/yyyy HH:mm' }}
{{ precio | currency:'COP':'symbol':'1.0-0' }}
{{ texto | uppercase | slice:0:50 }}
{{ objeto | json }}
{{ numero | number:'1.2-2' }}

<!-- async pipe: maneja suscripción -->
@if (usuarios$ | async; as lista) {
  @for (u of lista; track u.id) {
    <app-card [titulo]="u.nombre" />
  }
}

// Pipe personalizada
@Pipe({ name: 'truncar', standalone: true })
export class TruncarPipe implements PipeTransform {
  transform(val: string, len = 50): string {
    return val?.length > len ? val.slice(0, len) + '…' : val;
  }
}
@defer (Deferrable Views)Angular 17+
Carga componentes de forma diferida directamente en el template. Soporta triggers: on viewport, on idle, on interaction, when condicion. Mejora el LCP automáticamente.
💡 La forma más simple de lazy loading a nivel de componente. Sin código JavaScript adicional.
<!-- Carga cuando el elemento entra en pantalla -->
@defer (on viewport) {
  <app-comentarios />
} @loading (minimum 200ms) {
  <app-skeleton />
} @error {
  <p>Error al cargar comentarios</p>
} @placeholder {
  <div class="placeholder">Comentarios aquí</div>
}

<!-- Carga al interactuar con un elemento -->
<button #btn>Ver mapa</button>
@defer (on interaction(btn)) {
  <app-mapa-interactivo />
}
Guards & ResolversRouting
Los Guards protegen rutas (¿puede el usuario entrar?). Los Resolvers precargan datos antes de activar la ruta. En Angular 15+ son funciones simples, no clases con implementación de interfaz.
💡 canActivate, canDeactivate, canMatch y canLoad son los guards principales. Usa funciones, no clases.
// Guard moderno (función pura)
export const authGuard: CanActivateFn = () => {
  const auth   = inject(AuthService);
  const router = inject(Router);
  return auth.isLoggedIn()
    ? true
    : router.createUrlTree(['/login']);
};

// Resolver moderno
export const usuarioResolver: ResolveFn<Usuario> = (route) => {
  return inject(UsuarioService).getById(
    +route.paramMap.get('id')!
  );
};
ChangeDetection OnPushPerformance
Mecanismo que decide cuándo actualizar el DOM. Default revisa toda la app ante cualquier evento. OnPush solo revisa cuando cambia una referencia de @Input o un Signal emite. Mucho más eficiente.
💡 Siempre usa OnPush en producción. Con Signals + OnPush, Angular actualiza solo los nodos que cambian.
@Component({
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    <p>{{ nombre() }}</p>
    <p>Total: {{ total() | currency }}</p>
  `
})
export class OptimizadoComponent {
  nombre = signal('Angular');
  items  = signal<Item[]>([]);
  total  = computed(
    () => this.items().reduce((s, i) => s + i.precio, 0)
  );
}
ViewChild / viewChild()DOM
Accede a elementos del DOM o componentes hijos directamente desde la clase. @ViewChild es el decorador clásico. viewChild() (Angular 17+) retorna un Signal. Disponible solo después de ngAfterViewInit.
💡 Usa viewChild() con Signals. Es reactivo y no necesita el hook ngAfterViewInit.
@Component({ template: `<input #inp />` })
export class FormComponent {
  // Forma moderna con Signal
  inputEl = viewChild<ElementRef>('inp');

  // Acceso reactivo al elemento nativo
  enfocar() {
    this.inputEl()?.nativeElement.focus();
  }

  // Acceder a componente hijo
  modal = viewChild(ModalComponent);
  abrirModal() { this.modal()!.open(); }
}

💡 Úsalo para crear componentes contenedores reutilizables como cards, dialogs y layouts.

      <ng-content />
    </div>

export class CardComponent {}

<!-- Uso -->

Signals — Estado Reactivo Moderno

El sistema de reactividad de Angular 16+. Más simple y eficiente que RxJS para estado local.

⚡ Las 4 primitivas de Signals
signal(valor)
Estado mutable. Se lee llamándolo como función. Se actualiza con .set(), .update() o .mutate() para objetos.
const contador = signal(0);

contador();                   // leer → 0
contador.set(5);              // asignar
contador.update(n => n + 1); // transformar
computed(fn)
Valor derivado de otros signals. Solo recalcula cuando sus dependencias cambian. De solo lectura (asReadonly).
const precio = signal(100);
const iva    = signal(0.19);

const total = computed(
  () => precio() * (1 + iva())
);
total(); // → 119 (recalcula solo si precio o iva cambian)
effect(fn)
Efecto secundario que corre automáticamente cuando los signals que usa cambian. Para sincronización, logging o localStorage.
const tema = signal('dark');

effect(() => {
  // Se ejecuta cada vez que 'tema' cambia
  document.body.dataset.theme = tema();
  localStorage.setItem('tema', tema());
});
toSignal(obs$)
Convierte un Observable de RxJS a Signal. El puente entre el mundo reactivo. Gestiona la suscripción automáticamente.
import { toSignal } from '@angular/core/rxjs-interop';

export class ListaComponent {
  private svc = inject(DataService);

  datos = toSignal(this.svc.getAll(), {
    initialValue: []
  });
}
🏪 Signal Store — gestión de estado sin NgRx
// store/carrito.store.ts
@Injectable({ providedIn: 'root' })
export class CarritoStore {
  // Estado privado mutable
  private _items = signal<Producto[]>([]);

  // Estado público de solo lectura (exposición controlada)
  readonly items    = this._items.asReadonly();
  readonly total    = computed(() => this._items().reduce((s, p) => s + p.precio, 0));
  readonly cantidad = computed(() => this._items().length);
  readonly isEmpty  = computed(() => this._items().length === 0);

  // Acciones
  agregar(p: Producto) { this._items.update(items => [...items, p]); }
  eliminar(id: number) { this._items.update(items => items.filter(p => p.id !== id)); }
  vaciar()             { this._items.set([]); }
}

// En el componente — reactividad automática
export class CarritoComponent {
  store = inject(CarritoStore);
  // template usa store.items(), store.total(), etc.
}

@Directive({
  selector: '[appResaltar]',
  standalone: true,
  host: {
    '(mouseenter)': 'onEnter()',
    '(mouseleave)': 'onLeave()',
    '[style.cursor]': '"pointer"'

  onLeave() {
    this.renderer.removeStyle(this.el.nativeElement, 'background');
  }
}

// app.routes.ts
export const routes: Routes = [
  { path: '', redirectTo: 'home', pathMatch: 'full' },
  { path: 'home', component: HomeComponent },

  // Ruta con parámetro
  { path: 'usuario/:id', component: UsuarioComponent },

  // Lazy loading de componente standalone
  {
    path: 'perfil',
    canActivate: [authGuard],
    loadComponent: () => import('./perfil/perfil.component')
                         .then(m => m.PerfilComponent)
  },

  // Lazy loading de sub-rutas (feature routes)
  {
    path: 'admin',
    canActivate: [authGuard, roleGuard('admin')],
    loadChildren: () => import('./admin/admin.routes')
                        .then(m => m.ADMIN_ROUTES)
  },

  // Resolver — precarga datos antes de activar
  {
    path: 'detalle/:id',
    resolve: { producto: productoResolver },
    loadComponent: () => import('./detalle/detalle.component')
                         .then(m => m.DetalleComponent)
  },

  // Ruta comodín — 404
  { path: '**', component: NotFoundComponent }
];

export class DetalleComponent {

  queryParams = toSignal(this.route.queryParams);

Formularios Reactivos

Reactive Forms: el estándar para formularios complejos con validación robusta

📋 Reactive Forms completo con validación
export class RegistroComponent {
  private fb  = inject(FormBuilder);
  private svc = inject(AuthService);

  form = this.fb.group({
    nombre:   ['', [Validators.required, Validators.minLength(2)]],
    email:    ['', [Validators.required, Validators.email]],
    password: ['', [Validators.required, Validators.minLength(8),
                     passwordFuerteValidator]],
    edad:     [null, [Validators.min(18), Validators.max(120)]],
  });

  // Acceso rápido a controles
  get email()    { return this.form.get('email'); }
  get password() { return this.form.get('password'); }

  onSubmit() {
    if (this.form.invalid) { this.form.markAllAsTouched(); return; }
    this.svc.register(this.form.getRawValue()).subscribe();
  }
}

// Validador personalizado sincrónico
const passwordFuerteValidator: ValidatorFn = (c) =>
  /[A-Z]/.test(c.value) && /[0-9]/.test(c.value)
    ? null
    : { passwordDebil: true };

// Validador asíncrono (ej: verificar email en API)
const emailDisponibleValidator: AsyncValidatorFn = (c) => {
  return inject(AuthService).emailDisponible(c.value).pipe(
    map(disponible => disponible ? null : { emailTomado: true })
  );
};

HttpClient — Peticiones HTTP

CRUD, tipado genérico, interceptores y manejo de errores

🌐 Service con CRUD completo y manejo de errores
@Injectable({ providedIn: 'root' })
export class ProductoService {
  private http = inject(HttpClient);
  private url  = 'https://api.ejemplo.com/productos';

  getAll(params?: { pagina?: number; busqueda?: string }) {
    return this.http.get<Producto[]>(this.url, { params }).pipe(
      shareReplay(1),
      catchError(this.handleError)
    );
  }

  create(data: Omit<Producto, 'id'>) {
    return this.http.post<Producto>(this.url, data);
  }

  update(id: number, data: Partial<Producto>) {
    return this.http.patch<Producto>(`${this.url}/${id}`, data);
  }

  delete(id: number) {
    return this.http.delete<void>(`${this.url}/${id}`);
  }

  private handleError(err: HttpErrorResponse) {
    const msg = err.status === 0
      ? 'Error de red'
      : `Error ${err.status}: ${err.message}`;
    return throwError(() => new Error(msg));
  }
}

// Interceptor moderno (función)
export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const token = inject(AuthService).getToken();
  if (!token) return next(req);
  return next(req.clone({ setHeaders: { Authorization: `Bearer ${token}` } }));
};

🔍 Buscador con debounce + switchMap
export class BuscadorComponent {
  private svc = inject(ProductoService);

  termino   = signal('');
  cargando  = signal(false);
  errorMsg  = signal<string | null>(null);

  resultados = toSignal(
    toObservable(this.termino).pipe(
      debounceTime(300),
      distinctUntilChanged(),
      filter(q => q.trim().length >= 2),
      tap(() => { this.cargando.set(true); this.errorMsg.set(null); }),
      switchMap(q => this.svc.buscar(q).pipe(
        catchError(e => { this.errorMsg.set(e.message); return of([]); })
      )),
      tap(() => this.cargando.set(false))
    ),
    { initialValue: [] }
  );
}

Lifecycle Hooks

El ciclo de vida completo de un componente Angular

🔁 Ciclo de vida de un componente
constructor
Solo inyección de dependencias. Aquí va inject(). Nunca accedas al DOM ni hagas HTTP aquí.
ngOnChanges
Antes de ngOnInit y ante cada cambio de @Input. Recibe SimpleChanges con valores anteriores y nuevos.
ngOnInit
El más usado. Una sola vez tras inicializar @Inputs. Aquí va la lógica de arranque y peticiones HTTP iniciales.
ngDoCheck
Cada ciclo de detección de cambios. Muy costoso. Solo para detección manual en casos excepcionales.
ngAfterContentInit
Tras proyectar contenido con ng-content. Los @ContentChild están disponibles. Ejecuta una vez.
ngAfterViewInit
Tras renderizar la vista y componentes hijos. @ViewChild disponible. Interactúa con el DOM aquí.
ngAfterViewChecked
Tras cada verificación de vista. Muy costoso. Puede causar ExpressionChangedAfterItHasBeenChecked.
ngOnDestroy
Antes de destruir el componente. Desuscribe Observables, limpia timers y libera recursos para evitar memory leaks.
🧹 DestroyRef — evitar memory leaks (Angular 16+)
export class MiComponent {
  private destroyRef = inject(DestroyRef);

  ngOnInit() {
    // takeUntilDestroyed — se desuscribe solo al destruir el componente
    interval(1000).pipe(
      takeUntilDestroyed(this.destroyRef)
    ).subscribe(n => console.log(n));

    // También funciona fuera de ngOnInit en Angular 16+
    this.destroyRef.onDestroy(() => {
      console.log('Componente destruido — limpiando recursos');
    });
  }
}

// Alternativa: takeUntilDestroyed() en el campo de clase (Angular 16.1+)
export class Moderno {
  datos$ = inject(DataService).stream().pipe(
    takeUntilDestroyed()  // sin argumento: usa el DestroyRef del contexto
  );
}

Ecosistema de Herramientas

Las librerías y herramientas más importantes del mundo Angular

📦 Librerías esenciales del ecosistema
Angular Material
Biblioteca UI oficial de Google (Material Design 3). Botones, tablas, diálogos, formularios. Totalmente accesible y con theming con CSS custom properties.
Angular CDK
Kit de desarrollo de componentes. Drag & drop, virtual scrolling, overlays, portals, accessibility primitives. Base de Angular Material.
NgRx SignalStore
State management moderno con Signals. Mucho más simple que Redux. Ideal para apps medianas. El futuro del estado en Angular.
NgRx Store + Effects
State management estilo Redux. Store, Effects, Selectors, Entity. Para aplicaciones enterprise con lógica de estado compleja.
TanStack Query (Angular)
Caché de peticiones HTTP, estados de carga/error, invalidación y revalidación automática. Elimina gran parte del boilerplate de servicios HTTP.
Angular Fire
Integración oficial con Firebase. Auth, Firestore, Storage, Functions, Hosting. Ideal para prototipos y apps serverless.
PrimeNG
Suite completa de componentes UI. Tablas avanzadas, charts, tree, scheduler, calendar. Una de las más ricas del ecosistema Angular.
Tailwind CSS
Framework utility-first CSS. Compatible con Angular, fácil de configurar. Elimina la necesidad de CSS personalizado para la mayoría de casos.
Transloco
Internacionalización (i18n) moderna. Más flexible que i18n de Angular por defecto. Lazy loading de traducciones, múltiples idiomas simultáneos.
Nx (Nrwl)
Monorepo tools. Comparte código entre múltiples apps Angular, React, Node. Grafo de dependencias, affected commands. Para proyectos enterprise.
Jest + jest-preset-angular
Testing unitario rápido. Alternativa a Karma/Jasmine. Mejor experiencia de desarrollo, snapshots, coverage. Muy usado en proyectos modernos.
Cypress / Playwright
Testing E2E. Cypress tiene excelente integración con Angular. Playwright es más rápido y soporta múltiples navegadores. Component Testing disponible.

📅 Evolución de Angular — versiones clave
Angular 2 (2016)
Reescritura total. TypeScript, componentes, DI jerárquico. Ruptura total con AngularJS (1.x).
Angular 9 (2020)
Ivy compiler. Compilador por defecto. Bundles más pequeños, mejor debugging y tree-shaking.
Angular 14 (2022)
Standalone Components. inject() function. Typed Reactive Forms. El inicio del Angular moderno.
Angular 16 (2023)
Signals en developer preview. Required inputs. SSR mejorado. takeUntilDestroyed(). DestroyRef.
Angular 17 (2023)
Nueva sintaxis @if/@for/@switch. @defer. Signals estables. Standalone por defecto. Nueva web angular.dev.
Angular 18-19 (2024)
Signal inputs/outputs/queries. Zoneless (experimental). HMR mejorado. Material 3 estable. SSR con hydration.
💡 Angular sigue un ciclo de 2 versiones mayores por año (cada 6 meses). Política LTS de 18 meses. Angular 21 se esperaría en 2025.
